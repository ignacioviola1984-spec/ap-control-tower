Business Case

El valor no es “procesar más rápido”. Es controlar mejor dónde se usa el tiempo.

El sistema automatiza tareas repetibles y dirige la atención humana hacia datos faltantes, excepciones, fraude potencial y aprobación de pagos.

01

Capacidad operativa

Menos lectura, recarga y búsqueda manual. El equipo trabaja sobre una cola priorizada, no sobre el 100% de las facturas.

02

Riesgo antes del pago

Duplicados, desvíos contra OC, cambios de IBAN y diferencias cashflow–ERP se frenan antes de integrar un lote.

03

Gobierno y trazabilidad

Cada control, corrección, sign-off y decisión queda asociado a evidencia, responsable y timestamp.

04

Escala sin perder control

El motor aplica reglas explícitas y configurables; el gate humano continúa siendo obligatorio aun cuando crece el volumen.

Casos de uso

Un mismo control tower para las excepciones que realmente aparecen en AP.

≡

Duplicados y casi-duplicados

Número, proveedor, importe y proximidad temporal para frenar reenvíos y variaciones sospechosas.

⌁

Cambio de IBAN

Compara datos bancarios contra el maestro y bloquea transferencias con evidencia divergente.

✓

PO y non-PO

Match y saldo para ruta PO; aprobador, centro de coste y soporte para non-PO gobernada.

↗

Proformas y anticipos

Flujo separado que nunca entra al lote de facturas y alerta si falta la factura final.

↻

SEPA y tarjeta

Valida mandato o crea tareas de conciliación posterior sin forzar esos pagos dentro del lote.

±

Cashflow vs. ERP

Detecta diferencias antes del pago y evita que un error manual de Excel llegue al banco.

Seguridad de la información

Tus documentos se procesan para tu operación. No para entrenar modelos de Google.

El PoC combina decisiones de arquitectura propias con los compromisos contractuales y de seguridad de Google Cloud.

Compromiso Document AI
Google Cloud declara que no utiliza el contenido del cliente —incluidos documentos y predicciones— para entrenar sus modelos de Document AI.
Ver seguridad y uso de datos de Document AI ↗
Configuración actual del PoC Procesamiento temporal, sin repositorio de PDFs.

La aplicación procesa los documentos durante la sesión y no guarda una copia del PDF en una base de datos ni en un bucket. El sistema de archivos de Cloud Run opera en memoria y su contenido no persiste cuando la instancia se detiene.

Principios de control

Cuatro límites que el sistema no negocia.

01

Sin evidencia, no avanza

Las reglas hard bloquean y explican el motivo.

02

Una proforma no es una factura

Los anticipos mantienen un flujo propio.

03

Corregir no equivale a pagar

Toda modificación invalida sign-offs y obliga a revalidar.

04

El dinero lo libera una persona

No existe transición autónoma hacia el banco.
